## Входные данные

### `prolongations.csv`

Таблица проектов с информацией о факте пролонгации.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | int | ID проекта |
| `AM` | str | Имя менеджера по работе с клиентами |
| `month` | str | Месяц окончания (например, `"Январь 2023"`) |

> **Важно:** каждая пара `(id, month)` должна быть уникальной — дубликаты удаляются при загрузке.

---

### `financial_data.csv`

Финансовые данные по проектам — суммы отгрузок по месяцам.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | int | ID проекта |
| `<месяц>` | str / float | Сумма отгрузки за каждый месяц |

#### Специальные значения в ячейках:

| Значение | Интерпретация |
|----------|---------------|
| Числовое | Сумма отгрузки в рублях |
| `""` / `NaN` | Нет данных (не учитывается) |
| `"стоп"` / `"stop"` | Проект остановлен с этого месяца |
| `"в ноль"` / `"0"` | Проект активен, отгрузка = 0 руб. |

> `"в ноль"` — ключевое отличие от пустой ячейки: проект **ещё активен**, но не имел отгрузки в данном месяце. При расчёте суммы пролонгации берётся значение предыдущего месяца (рекурсивно).

### Ячейка 3 — Парсинг сумм `parse_amount(val)`

```python
def parse_amount(val, end=None) -> float | str | None
```

**Аргументы:**
- `val` — значение ячейки (строка или число)
- `end` — заглушка (не используется, оставлена для совместимости)

**Возвращает:**
- `"stop"` — если значение является маркером остановки

- `None` — если значение пустое или нераспознанное

Применяется к каждой колонке-месяцу в `financial_data.csv`.

### Ячейка 5 — Вспомогательные функции

#### `is_stopped(proj_id, month_col) → bool`

Проверяет, остановлен ли проект к указанному месяцу.

- `True` если проект не найден
- `True` если `stop_month <= month_col` (по позиции в `month_columns`)
- `False` иначе

---

#### `get_raw_shipment(proj_id, month_col) → float | None`

Возвращает «сырую» отгрузку проекта за месяц:

| Ситуация | Возвращает |
|----------|------------|
| Проект не найден | `None` |
| Проект остановлен в этом месяце | `None` |
| Месяц помечен «в ноль» | `0.0` |
| Отгрузка > 0 | `float` |
| Отгрузка == 0 (не «в ноль») | `None` |

---

#### `get_base_amount(proj_id, month_col) → float`

Возвращает базовую сумму для расчёта коэффициента.

**Логика:**
1. Если месяц помечен «в ноль» → рекурсивно берётся предыдущий месяц
2. Иначе → возвращается `get_raw_shipment(...)`, или `0.0` если `None`

---

#### `is_prolonged(proj_id, month_col) → bool`

Определяет, является ли проект пролонгированным в целевом месяце.

**Возвращает `True` если:**
- Проект активен (не остановлен)
- Месяц помечен «в ноль» **ИЛИ** отгрузка > 0

## Определения коэффициентов

### Коэффициент 1 (К1) — «Пролонгация первого месяца»

$$K1_M = \frac{\sum_{i \in \text{ended}(M-1)} \text{shipment}_i(M)}{\sum_{i \in \text{ended}(M-1)} \text{shipment}_i(M-1)}$$

- Числитель: сумма отгрузок в целевом месяце M для проектов, завершившихся в M-1
- Знаменатель: сумма отгрузок в M-1 для тех же проектов
- Учитываются только проекты, не остановленные к M-1

---

### Коэффициент 2 (К2) — «Пролонгация второго месяца»

$$K2_M = \frac{\sum_{i \in \text{ended}(M-2),\ \neg \text{prol}(M-1)} \text{shipment}_i(M)}{\sum_{i \in \text{ended}(M-2),\ \neg \text{prol}(M-1)} \text{shipment}_i(M-2)}$$

- Берутся проекты, завершившиеся в M-2, которые **НЕ** были пролонгированы в M-1
- Проверяется их пролонгация в целевом месяце M
